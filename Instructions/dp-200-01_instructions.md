# DP 200 - データ プラットフォーム ソリューションの実装
# ラボ 1 - データ エンジニアのための Azure

**推定時間**: 60 分

**前提条件**: この課題のケース スタディを既に読んでいることを前提にしています。

**ラボ ファイル**: このラボのファイルは、_Allfiles\Labfiles\Starter\DP-200.1_のフォルダーにあります。

## ラボの概要

学生は、レッスンとケース スタディで得られた情報をもとに、AdventureWorks 内のデジタル環境の変革プロジェクトの成果物を詳しく調べます。まず、データ使用の進化が組織にどのような新しい機会を与えたかを特定します。また、ビジネス ニーズに対応するために使用される Azure Data プラットフォーム サービスについても説明し、データ エンジニアが実行するタスクを定義します。最後に、学生は AdventureWorks の Data Engineering の成果物を完成させます。

## 課題の目的
  
この課題を完了すると、次のことができるようになります。

1. AdventureWorks 内の進化するデータの世界を特定する。
2. AdventureWorks に使用する Azure Data プラットフォーム サービスを決定します。
3. データ エンジニアが実行するタスクを特定する。
4. AdventureWorks の Data Engineering 成果物の最終処理を行います。

## シナリオ
  
あなたは、デジタル変革プロジェクトに関して AdventureWorks の経営陣と従業員に助言するために、シニア データ エンジニアとして採用されました。  AdventureWorks は、10 年以上にわたり、最終消費者と流通業者に、自転車および自転車部品を直接販売してきました。ここ数年、同社はさまざまな業界がどのようにクラウド技術の新発展を利用して、よりパーソナライズされた魅力的なサービスを顧客に提供しているかを観察してきました。同社は業界をリードしたいと考えています。

あなたは、IT 部門と連携して、組織でディスカバリー ワークショップを実施し、AdventureWorks と顧客の両方のために使用できるデータ プラットフォーム テクノロジーを特定するよう依頼されました。彼らは特に、このテクノロジによりさまざまなレベルで、ビジネスを顧客に近づけたいと考えています。これには、購入するとき、または顧客サービスを利用するときに個人別にカスタマイズされたオファーを作成して、使用される自転車に関する利用統計情報を提供することが含まれます。また、Data Warehouse に保持されている既存のレポート システムを管理する必要もあります。

最初のステップでは、クラウド テクノロジの現状を評価し、次の分析を実行してプロジェクトを実行できることを確認します。

1. AdventureWorks 内の進化するデータの世界が確認されました
2. AdventureWorks で使用する Azure Data プラットフォーム サービスを決定した。
3. データ エンジニアが実行するタスクを特定する。
4. AdventureWorks のデータ エンジニアリングの成果物を完成させました
